riid.data.labeling module¶

This module contains utility functions for managing ground truth information.

riid.data.labeling.isotope_name_is_valid(isotope: str)¶

Validates whether or not the given string contains a properly formatted radioisotope name.

Note that this function does NOT look up a string to determine if the string corresponds to a radioisotope that actually exists, it just checks the format.

The regular expression used by this function looks for the following (in order): - 1 capital letter - 0 to 1 lowercase letters - 1 to 3 numbers - an optional “m” for metastable

Examples of properly formatted isotope names: - Y88 - Ba133 - Ho166m

Parameters:

isotope – a string containing the isotope name

Returns:

A bool representing whether or not the name string is valid.

riid.data.labeling.label_to_index_element(label_val: str, label_level='Isotope', verbose=False) → tuple¶

Tries to map a label to a tuple for use in DataFrame MultiIndex columns.

Depending on the level of the label value, you will get different tuple: |`label_level` |Resulting Tuple | |--------------|————————–| |Seed |(Category, Isotope, Seed) | |Isotope |(Category, Isotope) | |Category |(Category,) |

Parameters:

  • label_val – The part of the label (Category, Isotope, Seed) from which to map the other two label values, if possible.

  • label_level – The level of the part of the label provided, e.g. “Category”, “Isotope”, or “Seed.”

Returns:

An n-tuple containing the Category, Isotope, Seed values identified

for the old label format.
